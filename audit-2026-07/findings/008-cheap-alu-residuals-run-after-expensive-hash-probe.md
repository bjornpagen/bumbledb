## Cheap ALU residuals run after expensive hash probes in every join node — the temporal filters on the ring lanes buy nothing

category: perf | severity: high | verdict: CONFIRMED | finder: perf:rings

### Summary

Within a join node, the vectorized executor runs every sibling hash probe (memory-bound: hash + bucket loads over multi-MB COLT maps) before evaluating the node's residual comparisons (pure ALU: whole-value compares, word compares, the vectorized Allen configuration kernel). The residual operands provably never depend on sibling probe results, so hoisting the residual compaction blocks ahead of the sibling-probe loop is a pure ordering change with identical semantics — and on the ring lanes it would kill 50-99.4% of the closing probes before they run. The night-run pins show the symptom directly: `r1_wash_ring` (62,523.5 µs) and `r2_temporal_ring` (62,857.3 µs) clock identically even though r2 adds three ~0.6%-selective Allen INTERSECTS conditions — the temporal filters currently buy nothing because the probes dominate and run first.

### Evidence (all verified against the code)

- **Ordering.** `crates/bumbledb/src/exec/run/probe_pass.rs:56` — the sibling-probe loop (`for sub_idx in 0..node.subatoms.len()`, hash :65-182, prefetch :186-200, probe+compact :201-234) — precedes the residual phase at `probe_pass.rs:239` (`counters.phase_start(node_idx, JoinPhase::Residual)`), which holds the whole-value (:240-263), word (:267-290), Allen (:298-334), and duration (:338-376) residual blocks. The line-parallel twin `run_node.rs:182` (sibling loop) / `run_node.rs:276-277` ("Residuals run as batch survivor compaction after the probes") has the same order; the twin discipline (a change here needs its mirror there) is recorded at `run_node.rs:164-173`.
- **Independence (the reorder is safe).** The residual operand closures read only cover batch words and outer bindings: `probe_pass.rs:249-252` (`scratch.entry_keys[element*arity + word + offset]` / `scratch.pending_bindings[parent*slot_count + slot + offset]`), likewise :279-282, :313-316, :352-355. The sibling-probe loop writes only `hashes`, `probe_keys`, `mask`, `sibling_children`, and compacts `survivors` — never `entry_keys` or `pending_bindings`. Sibling probes bind no variables by construction: `plan/fj/binary2fj.rs:40-44` filters a probe subatom's vars to those already `available`. `sibling_children` is indexed by stable element index (`probe_pass.rs:229`), and the final survivor set is the same intersection either way, so the routing arm (`assemble`, :509-528) reads only entries that were written.
- **Plan shape on the ring lanes.** `binary2fj` (the paper's Fig. 7 transcription) yields node0 = [T0 cover, T1(v1) probe], node1 = [T1(v2,v5) cover, **T2(v2,v0) probe**], node2 = [T2(v6)] for the order [T0,T1,T2]. Residuals attach to the earliest node with both sides bound (`plan/fj.rs:242-244`; `docs/architecture/40-execution.md:124-127`): `lt(v0,v2)` (r1 and r2, `scenarios/rings.rs:199-200`) and `intersects(v4,v5)` (r2, `rings.rs:202`) both land at node1 — the same node as the 2-word T2 closing probe, which runs first.
- **Selectivity.** `scenarios/rings/corpus.rs:92-96`: span width 1..172,800 s uniformly placed over a ~29.8M-s horizon → pairwise INTERSECTS passes ≈ 0.6% of random pairs. `lt(v0,v2)` passes ≈ 50%.
- **Bench symptom.** `bench-out/night-2026-07-20/scenarios/scenarios.md:53-55`: r1 ours p50 = 62,523.5 µs; r2 ours p50 = 62,857.3 µs — statistically identical despite r2's added filters. (r4's 5.7e7 closing probes at 1.577 s give ~28 ns per probe-dominated element — consistent with node1's multi-million-element 2-path stream dominating both lanes.)
- **No measurement recorded.** `docs/architecture/40-execution.md:125-127` states the order ("iterate cover → probe siblings → evaluate the node's residuals"; "residuals run as batch survivor compaction after the probes") with no ablation gravestone or refusal — in files where every other ordering/batching decision carries a measured verdict (e.g. the survivor-routing refusal at `probe_pass.rs:447-460`). The Free Join paper is silent by design: `docs/free-join-paper/.../tex/02-background.tex:23-25` assumes selections pre-pushed to base tables and has no cross-atom residual concept, and `40-execution.md:139-141` records exactly this deviation — "we own filter placement because there is no external optimizer." Owning placement without a cost-ordering rule is the gap.

### Bench impact

- **r2_temporal_ring**: hoisting node1's residual block runs `lt(v0,v2)` (50%) and `intersects(v4,v5)` (~0.6%) before the T2(v2,v0) probe, cutting closing probes by ~99.7%. The lane then pays cover iteration + the 4-endpoint gather + the vectorized Allen classify over the 2-path stream instead of a hash probe per element. Directionally a multi-x win; the finder's ~10-15 ms projection is an estimate (the gather/classify cost over the full stream remains), not a measurement.
- **r1_wash_ring**: `lt(v0,v2)` first halves the closing probes — a smaller but free win.
- Every lane whose plan places residuals at a node that also carries sibling probes (t-lanes, j-lanes with range conditions) inherits the same reordering for free; lanes without node-placed residuals are untouched.
- Worst case of the reorder: a residual that passes everything costs one extra ~1-2 ns ALU pass over the pre-probe survivor set — bounded and small against the 5-30 ns probes it can never make more numerous.

### Suggested fix

Move the whole-value, word, Allen, and duration residual compaction blocks ahead of the sibling-probe loop in `probe_pass.rs` and its line-parallel mirror in `run_node.rs` (the recorded twin discipline — both or neither). Membership point probes and anti-probes stay where they are (they read `sibling_children` / probe COLTs and belong in the probe cost class). The representation-first version, per `docs/design/representation-first.md`: make the node's rejection pipeline an ordered list of rejection steps sorted by cost class at plan time (ALU residuals → hash probes → point/anti probes) instead of a fixed structural sequence — the ordering becomes data, and the "which filter first" branch disappears. Either way, run the interleaved A/B against the ring lanes and record the verdict where the doc currently states the order bare (`40-execution.md:127`).
