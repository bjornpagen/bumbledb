## Adversarial notes (the mandate's named targets)

- **Predecessor seam = Disjoint's spend site: CONFIRMED.** The ≥ probe's filter (`starts_with(seek)`, judgment.rs:709-711) admits only `start == s` exactly; lexicographic order guarantees any key between `group‖s` and `group‖s‖e` shares the prefix, so the probe cannot skip an exact-start entry. The predecessor branch's conviction — "a predecessor that has ended proves nothing covers `s`" (judgment.rs:704-707) — and the exact-start branch's silent skip of earlier group keys are each sound *only* because two segments overlapping `s` would violate Disjoint; off-premise both can only under-cover (reject), matching `sweep_never_false_accepts`'s one-sidedness. The model's seam note (Sweep.lean:24-34, 40-48) is accurate to the line.
- **Empty segment group**: `located = None` → `entry = None`, `chain = None` → empty iterator → `maximal(s, s)` → reject (judgment.rs:726-758); Lean `sweepFrom e s [] = decide (e ≤ s) = false` since `s < e` by `Interval.h`. Match; pinned by `empty_input_packs_nothing_and_covers_nothing` (sweep.rs:316-319).
- **First-segment-starts-late gap**: no exact-start hit, predecessor absent or ended → empty walk → reject, without ever touching the later segments; Lean's fold reaches the same `false` via the gap branch on the first segment. Verdicts equal; consumption differs only below the fold's mechanism fence.
- **Prefix-group boundary**: `DeterminantSegments` ends the walk at the first key outside `group` (judgment.rs:834-837) — the fold's list end; an in-group key of wrong length is a hard corruption error, never a silently skipped segment (judgment.rs:838-846). No boundary leak found.
- **Ray targets**: a bounded segment after a ray never exceeds the frontier (sweep.rs:303); a source ray demands `frontier == MAX`, reachable only through a target ray — `ray_needs_ray` line for line.

## Divergences

1. **(b)** Per-consumed-segment σ re-check: `GapAt::segment → check_segment` (judgment.rs:764-776, 864-866) makes `check_coverage`'s verdict `coverage ∧ (every consumed segment satisfies σ)`; `sweepCovered` (Sweep.lean:129) models pure coverage, and Sweep.lean's fold does not determine the σ conjunct (acknowledged as a hook in sweep.rs:35-37 and Sweep.lean's `sweepFrom` doc silence). σ semantics belong to Dependencies (pairing #3); recorded here as a spec gap of *this* file's model only. Severity: low.
2. **(b)** Degenerate window: Rust declares it "vacuously covered" (sweep.rs:57-58; `run=(s,s)`, `frontier >= e` fires immediately); Lean cannot state it — `Interval` carries `start < end`. Unreachable through `check_coverage` (probe intervals are acceptance-gated valid intervals). Severity: negligible.
3. **(b)** Windowed gap verdict is delegated: if a windowed continuation's `maximal` returned `Ok`, `sweep` returns `Ok(())` — an accept-on-gap (sweep.rs:96-99) — where `sweepFrom` hard-codes `false`. The only windowed continuation, `GapAt`, always errs (judgment.rs:868-870), so the divergence is unreachable; the spec simply doesn't determine `sweep` for a non-convicting windowed caller. Severity: negligible.
4. **(b)** Pack's production sort is `sort_unstable` on `[start, end]` lexicographic pairs (finalize.rs:37-41); the spec's `sortByStart` is insertion sort by start alone (Query/Aggregates.lean:314-316), so `pack_is_the_sweep` (Sweep.lean:342) literally covers a possibly different tie order among equal starts. Closed by composition: Rust's output is `Ordered`, `sortByStart` is the identity on it, and `pack_canonical` + `pack_extensional` force a unique canonical result — but the transfer is an argument, not a stated theorem. Severity: low; a one-line lemma (sweepRuns invariant under start-ordered permutation) would close it.
