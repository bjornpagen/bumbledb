## Divergences (dual-cited, classed)

- **F1 — class (a).** `rejection_is_complete` (Txn.lean:279-297) proves the rejection set contains EVERY violated statement of the final state, over all of `T.statements` — which includes both `functionality` and `containment` forms (`Schema.lean:242-248`). Rust: key violations detected in phase 2 preempt the containment judgment entirely — `commit/apply.rs:54-58` seals and returns only the violated KEY statements; `judgment.rs::judge` never runs (`commit/write.rs:70-75`: "every violated key statement ... **or** every violated containment statement"). A final state violating both a key and a containment rejects with the containment citations missing. The behavior is documented (`docs/architecture/30-dependencies.md:68-74`, "never a mix" — while citing this very theorem), but the Txn.lean narrowing record (lines 56-62) explicitly claims "membership completeness" is KEPT, so the Lean-normative reading is not narrowed to per-class completeness. Severity: error-payload only — no invalid state ever commits; the accept path is unaffected. Disposition candidates: record the per-class narrowing in Txn.lean, or weaken the theorem to per-class completeness, or run the containment judgment even under key failure.
- **F2 — class (b), recorded.** The Rust write surface has failure modes outside the three-constructor `WriteResult`: `ForeignSnapshot` (write.rs:93-95), `FreshExhausted`, `Lmdb`/`Io`/`CommitSync` (bounded retry, `commit/write.rs:36-58`), closure errors, and the non-reentrancy panic (write.rs:113-117). Recorded: "models the protocol, not the environment" (Txn.lean:66-68); durability is a covenant refusal (Txn.lean:10-12).
- **F3 — class (b), recorded.** `bulk_load` chunking (write.rs:188-226) vs `scanLoad`'s ONE final-state judgment (Txn.lean:537-539): Rust judges every 4096-fact prefix, so a whole-valid load with an invalid prefix rejects, and a failing chunk leaves prior chunks committed (`BulkLoadError.committed`, write.rs:215-218) where the model's reject lands nothing. Both faces are recorded (Txn.lean:69-77: a chunked load IS a sequence of commits, each judged; recipe 28's first law is host-facing). The narrowing is honest: each chunk is a genuine `Db::write` with full judgment.
- **F4 — class (b), recorded-adjacent.** `flush_escaped_fresh_ids` (commit/write.rs:181-201) persists `Q` high-water marks with no generation bump and no `commit_seq` advance — two states with equal generation can differ in fresh-allocation floor, and `write_from` accepts a witness taken before such a flush. Unobservable to any read or judgment (no query reads `Q`); fresh-identity allocation is the recorded not-modeled mechanism (Txn.lean:75-77).
- **F5 — class (b).** `Generation` is modeled with decidable equality only (Txn.lean:301-307); mechanism has arithmetic — `GenerationId::next` (`storage/env.rs:72`) advances it, and `GenerationId(u64)` (env.rs:47) is host-visible via `Db::generation` (maintain.rs:71). The PROTOCOL spends only `!=` (write.rs:138); the narrowing ("the protocol is one compare, never arithmetic") reads as protocol-scoped and holds.
- **F6 — class (b).** Fact identity in the delta and store is the 32-byte `fact_hash` (`delta.rs:96`; insert.rs:45); the model identifies facts extensionally. Hash-collision freedom is an unmodeled ambient premise of the whole storage layer, not specific to Txn.

## GRADE: B

Justification: the lifecycle machinery is a close, adversarially-checked match — the delta is genuinely a net set pair whose only order sensitivity is the final state itself; the generation compare is exactly the modeled one `if`, type-disjoint from dependency failure, correctly blind to no-op commits; snapshot reads factor through one pinned txn including across the parked-reader optimization; bulk_load's chunking is precisely the recorded sequence-of-commits narrowing. The grade is not an A because of F1: a reachable input class (key + containment violated in one final state) produces a rejection payload the normative theorem forbids, and the Lean narrowing record affirmatively claims the completeness that is lost — a spec/implementation contradiction in the theorem's own citation chain, even though it is payload-only and documented on the Rust side.
