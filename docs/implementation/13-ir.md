# PRD 13 — The Query IR

Authority: `docs/architecture/20-query-ir.md` (IR shape — normative; Value enum;
no-wildcard representation).

## Purpose

The pure-data query IR: the engine's public query contract. Data only — no methods
beyond constructors/accessors, no builder.

## Technical direction

- `ir` module, faithful to the doc's normative sketch: `Query { finds, atoms,
  predicates }`, `Atom { relation: RelationId, bindings: Vec<(FieldId, Term)> }`,
  `Term::{Var(VarId), Param(ParamId), Literal(Value)}`,
  `Value::{Bool(bool), U64(u64), I64(i64), Enum(u8), String(Box<[u8]>),
  Bytes(Box<[u8]>)}` — exactly six, no universal integer,
  `FindTerm::{Var(VarId), Aggregate { op: AggOp, over: Option<VarId> }}`,
  `AggOp::{Sum, Min, Max, Count}` (Count carries `over: None` by the doc — enforce in
  PRD 14, representable here), `Comparison { op, lhs, rhs }`,
  `CmpOp::{Eq, Ne, Lt, Le, Gt, Ge}`.
- `VarId(u16)`, `ParamId(u16)` dense newtypes. All types `Clone + PartialEq + Debug`;
  fields `pub` (it is *data*); `#[non_exhaustive]` on nothing (single-owner project,
  the IR is intentionally unstable).
- No `Display`, no builders, no serde (add only when a reader exists — rule 3).

## Non-goals

Validation (PRD 14). Any semantic behavior.

## Passing criteria

- Unit tests: construction of the doc's example query shapes (point lookup, FK-walk
  join, aggregate-by-account, zero-binding atom) as literals — these double as
  documentation; `Value` covers every `10-data-model.md` type (a test constructs one
  of each — the anti-Bytes-hole assertion, post-mortem §13).
- Global commands green.
